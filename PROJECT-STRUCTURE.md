# 项目文件夹说明

这个仓库当前包含多个子项目和配套文件。下面按根目录文件夹说明它们各自的职责，方便后续开发、部署和协作时快速定位。

## 根目录

- `frontend/`
  - 这是原有的主前端项目。
  - 从当前代码结构看，它更偏向“后台管理系统 / 业务管理端”，包含商品、订单、用户等管理页面。
  - 适合继续承载内部运营、管理后台一类功能。

- `portal-ai-frontend/`
  - 这是当前整理出来的前端项目。
  - 它现在被拆成“公司门户官网 + AI 平台”两部分：
  - `public` 部分负责官网展示。
  - `login` 部分负责 AI 平台登录。
  - `ai` 部分负责登录后的 AI 工作台。
  - 这个项目更适合作为对外官网和 AI 入口的前端。

- `mcx-system/`
  - 这是后端主服务项目。
  - 从代码和 `CLAUDE.md` 看，它是 Rust 编写的 API 服务，负责业务接口、认证、数据库访问和支付等能力。
  - 它和前端项目配合使用，提供数据和登录接口。

- `deploy/`
  - 部署脚本和发布辅助文件目录。
  - 用于前后端发布、服务器部署、打包流程等。

- `docs/`
  - 项目文档目录。
  - 用于保存设计说明、方案说明、接口说明或其他补充文档。

- `.idea/`
  - JetBrains 系列 IDE 的本地工程配置目录。
  - 一般只影响本地开发环境，不属于业务代码。

## 根目录文件

- `deploy.ps1`
  - 根目录发布入口脚本。
  - 当前仓库的发布流程通常会优先看这里。

- `CLAUDE.md`
  - 当前仓库的协作说明和项目背景说明。
  - 里面记录了项目结构、部署约定和工作方式。

- `.gitignore`
  - Git 忽略规则文件。
  - 用于排除构建产物、缓存和临时文件。

## 当前前端项目内部结构

### `portal-ai-frontend/src/`

- `layouts/`
  - 页面布局层。
  - `PublicLayout.vue`：公司门户官网的公共布局。
  - `AIAppLayout.vue`：AI 平台登录后的应用布局。

- `router/`
  - 路由定义目录。
  - 负责区分官网路由、登录路由和 AI 平台受保护路由。

- `stores/`
  - Pinia 状态管理目录。
  - `auth.js`：当前用于模拟登录状态和本地存储。
  - `index.js`：Pinia 入口。

- `styles/`
  - 全局样式目录。
  - 负责官网和 AI 平台统一视觉风格。

- `views/public/`
  - 公司门户官网页面。
  - `HomeView.vue`：门户首页。
  - `PortalView.vue`：官网介绍页。

- `views/auth/`
  - 登录相关页面。
  - `LoginView.vue`：AI 平台登录页。

- `views/ai/`
  - AI 平台页面。
  - `AIPlatformView.vue`：AI 平台首页。
  - `CoursesView.vue`：AI 课程中心。
  - `ToolsView.vue`：AI 效率工具页。

- `views/errors/`
  - 错误页面目录。
  - `NotFoundView.vue`：404 页面。

## 当前建议的职责边界

- `frontend/` 继续放管理后台。
- `portal-ai-frontend/` 继续放公司门户官网和 AI 平台前端。
- `mcx-system/` 继续放后端 API 和业务服务。
- `deploy/` 和 `deploy.ps1` 继续放发布与部署流程。

- 如果后续你希望把“公司门户官网”和“AI 平台”拆成两个独立部署目标，也可以继续在 `portal-ai-frontend/` 内再做更细的入口拆分。
