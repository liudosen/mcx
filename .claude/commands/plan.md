# Plan Command

## Description
Create a structured implementation plan before writing code.

## When to Use
- Implementing new features
- Making architectural changes
- Adding new API endpoints
- Database migrations

## What It Does
1. Analyzes the request thoroughly
2. Defines the scope and acceptance criteria
3. Designs the API contract (if applicable)
4. Plans the implementation steps
5. Identifies potential risks and error cases

## Usage
`/plan <description of what to build>`

## Output
A structured plan with:
- **Overview**: What we're building
- **API Contract**: Request/response shapes
- **Implementation Steps**: Numbered list
- **Error Handling**: What can go wrong
- **Testing Strategy**: How to verify
